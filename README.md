<h1 align="center">DataRecovery</h1>

<p align="center">
  使用 C#、.NET 9 和 Avalonia 构建的跨平台数据恢复与文件级安全删除工具。
</p>

## 项目简介

DataRecovery 面向 U 盘、移动硬盘、本地 SSD/HDD 和原始磁盘镜像，提供设备选择、扫描、候选文件筛选以及安全恢复工作流。界面采用三步恢复向导，支持同时选择多个数据源，并明确区分“已删除文件”和“原路径未知的特征扫描候选”。文件粉碎是独立的主动写入功能，必须明确选择文件、阅读说明并勾选同意后才会运行。

> DataRecovery 当前处于可运行的 MVP 阶段。重要数据应首先制作整盘镜像，源设备应立即停止写入。对于物理损坏、异响或大量坏扇区的设备，请优先交由专业数据恢复机构处理。

## 主要功能

- 枚举 U 盘、移动硬盘、本地 SSD/HDD 和已挂载卷。
- 同时勾选多个磁盘或镜像，依次执行扫描。
- 打开 `.img`、`.dd`、`.raw`、`.bin`、`.iso` 磁盘镜像。
- 识别 FAT12、FAT16、FAT32、exFAT、NTFS、Ext2 和 Ext3。
- 解析 FAT12/16/32 删除目录项、短文件名/LFN 和仍可用的簇链。
- 解析 exFAT 删除目录项，恢复仍保留元数据的连续文件。
- 解析 NTFS `$MFT` 删除记录，支持驻留数据及未压缩、未加密的非驻留 Data Runs。
- 解析 Ext2/Ext3 未分配 inode，支持直接块、一级和二级间接块。
- 使用统一多区段恢复模型，按逻辑顺序重组碎片簇、Data Runs、Ext 数据块及稀疏区段。
- 不依赖旧文件系统，按结构查找 JPEG、PNG、GIF、BMP、WebP、PDF、ZIP/Office、7-Zip、PE/EXE/DLL、ELF、WAV、AVI 和 SQLite 等候选数据。
- 快速格式化后若旧 exFAT 备份引导区仍存在，可额外恢复格式化前的活动/删除目录项；不存在时仍会继续通用结构扫描。
- 使用全部可用逻辑 CPU 进行并行分析。
- 采用单路顺序读盘、8 MB 数据块和 64 KB 边界重叠，兼顾机械硬盘与 SSD。
- 扫描结果实时分批显示，支持进度、取消、分类和搜索。
- 支持照片、文档、压缩包、程序及其他类型筛选。
- 同名恢复文件自动改名，不覆盖目标目录已有文件。
- 多设备结果记录各自源盘，恢复时自动从正确的数据源读取。
- 顶部设置菜单可选择扫描范围，默认使用“已删除文件”。
- 源盘以只读共享方式打开，扫描过程不会主动写入源设备。
- 独立的文件粉碎工具支持 1/3/7 次覆盖、逐遍落盘、最终 SHA-256 读取校验、随机改名、截断和删除。
- 粉碎前批量预检文件、父目录和独占访问条件；拒绝目录、链接/重解析点、稀疏、压缩、加密及网络文件，并在 Windows 上拒绝硬链接。

## 扫描类型

| 扫描类型 | 说明 | 适用场景 |
| --- | --- | --- |
| 已删除文件（默认） | 读取 FAT 目录项、NTFS MFT、Ext inode 等删除元数据 | 希望尽量保留原文件名、大小和路径信息 |
| 格式化/丢失文件 | 不依赖旧文件系统，扫描原始扇区中的文件结构 | 分区被快速格式化，或目录元数据已经损坏/丢失 |
| 全部类型 | 组合删除元数据与文件特征扫描 | 希望进行更全面但耗时更长的扫描 |

“格式化/丢失文件”不要求知道格式化前使用 FAT、exFAT、NTFS 还是 Ext。它表示扫描器在原始数据中识别出文件结构，但通常无法确定原文件名和原始路径；候选内容也可能是缩略图、旧数据或其他文件中的嵌入资源。

### 格式化后的磁盘

1. 立即停止向该磁盘写入新数据。
2. 在顶部“设置”中选择“格式化/丢失文件”；需要同时查找当前文件系统删除项时选择“全部类型”。
3. 勾选源磁盘后执行完整扫描，并把恢复结果保存到另一块磁盘。

通用结构扫描与格式化前的文件系统无关。若旧目录元数据仍存在，扫描器会尽量保留名称和大小；若元数据已被覆盖，则使用 `恢复候选_偏移量.扩展名` 命名。结构扫描按连续数据恢复，原文件若已碎片化，结果可能不完整。

## 文件级安全删除

顶部“文件粉碎”入口只处理用户逐个选择的普通文件，不接受目录。执行流程为：

1. 对整批文件进行非破坏性预检，确认文件类型、独占访问、父目录改名/删除权限及恢复源冲突。
2. 按所选模式执行 1、3 或 7 次覆盖，每一遍均请求落盘。
3. 重新读取最终覆盖结果并进行 SHA-256 校验。
4. 随机化当前目录项名称，截断文件并删除目录项。

默认使用单次随机覆盖。3 次和 7 次模式会显著增加耗时和写入量，并不表示对所有介质有更高保证。开始前必须阅读风险说明并勾选同意；文件列表或覆盖模式变化后需要重新确认。

> 文件级覆盖只适合传统机械硬盘上的普通、非写时复制文件。SSD、U 盘、存储卡等闪存设备可能因磨损均衡和备用块保留旧数据；APFS、Btrfs、ZFS、ReFS、快照、日志、云端版本和备份也可能保留副本。本功能不承诺物理介质上的绝对清除。需要介质级清除时，应使用设备厂商提供的 Sanitize、Secure Erase 或加密擦除方案。

## 文件系统支持状态

| 文件系统 | 格式识别 | 删除元数据恢复 | 文件特征扫描 |
| --- | :---: | :---: | :---: |
| FAT12 / FAT16 / FAT32 | ✅ | ✅ 短名/LFN、簇链 | ✅ |
| exFAT | ✅ | ✅ 连续文件 | ✅ |
| NTFS | ✅ | ✅ MFT 驻留/非驻留数据 | ✅ |
| Ext2 / Ext3 | ✅ | ✅ inode、直接/一二级间接块 | ✅ |

FAT 删除后若簇链已被清零，扫描器只能对连续且仍为空闲的簇进行保守推断；原本碎片化的清零链无法可靠重建。NTFS 会跳过压缩、加密及依赖 `ATTRIBUTE_LIST` 的文件。Ext inode 不保存原文件名，因此候选文件使用 inode 编号命名；Ext3 只读取当前磁盘元数据，不重放 journal。

## 安全原则

1. 删除文件后立即停止使用源设备。
2. 不要将 DataRecovery 安装到丢失数据所在磁盘。
3. 不要把恢复结果保存回源磁盘。
4. 优先扫描磁盘镜像，而不是反复读取故障设备。
5. Windows 原始卷通常需要管理员权限；Linux/macOS 设备节点通常需要 root 或磁盘读取权限。
6. 不要粉碎仍位于恢复源上的文件；程序会对当前已选择的恢复源执行冲突检查。

```powershell
dotnet restore
dotnet run --project src/DataRecovery.App
```

## 已知限制

- 文件特征扫描无法可靠恢复原文件名、原目录和时间戳。
- 特征候选可能包含误报、缩略图或文件内部嵌入资源。
- 文件碎片化、覆盖、压缩和加密会降低恢复成功率。
- 快速格式化通常只覆盖文件系统元数据；全格式化、TRIM/Discard 或后续写入可能已经清除实际内容。
- FAT 8.3 删除项的首字符会丢失；若没有可用 LFN，名称首字符以 `_` 代替。
- NTFS 暂不恢复压缩、加密、`ATTRIBUTE_LIST` 扩展记录及完整父目录树。
- Ext2/3 暂不重放 journal 或恢复原目录项名称，也不解析三级间接块。
- 原始镜像当前要求文件系统从偏移 0 开始，尚不自动解析 MBR/GPT 分区起始位置。
- 大于当前扫描块且文件尾不在同一数据块中的候选可能被标记为“部分”。
- SSD 执行 TRIM 后，被删除的数据可能已经清零，通常无法恢复。
- 暂不支持 RAID 重组、坏扇区重试、NTFS 压缩/加密、Ext 日志重放和完整目录树重建。
- 文件粉碎不能清除 SSD/闪存控制器保留块、文件系统快照、日志、云端版本或备份中的副本。
- Unix 平台的硬链接检测能力取决于运行环境；对来源不明或需要合规销毁的数据，应改用整盘加密擦除或厂商安全清除工具。

## 开发路线

- 自动解析 MBR/GPT 分区并扫描镜像内的多个卷。
- 改进 FAT 已清零碎片簇链的内容关联与校验。
- 扩展 NTFS `ATTRIBUTE_LIST`、压缩文件和父目录重建。
- 增加 Ext 目录项残留、三级间接块与 Ext3 journal 辅助恢复。
- 碎片文件重组和文件内容校验。
- 磁盘镜像制作、断点续扫和坏扇区策略。
- 扫描会话保存及结果导出。
